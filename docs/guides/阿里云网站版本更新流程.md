# Final Killer 阿里云网站版本更新流程

这份流程用于网站已经成功部署到阿里云之后，后续发布新版本、更新题库、排查异常。

核心原则：

- 本地开发和测试通过后，先推送到 GitHub。
- 服务器只从 GitHub 拉取代码，不在服务器上手改业务代码。
- 生产环境已有用户数据后，不要再执行 `npm run seed`。
- 每次正式更新前先备份 SQLite 数据库。

## 一、更新前先在本地完成

在本地项目目录执行：

```powershell
cd "D:\项目\Final Killer"
git status --short
npm test
npm run build
```

如果测试和构建都通过，再提交并推送：

```powershell
git add .
git commit -m "描述本次更新内容"
git push origin main
```

注意：

- 不要把 `.env`、数据库文件、日志文件提交到 GitHub。
- `.env.example` 可以提交，它只放示例配置。
- 如果只是题库内容更新，优先走管理员后台上传 JSON，不要为了题库更新改 `data/questions.json` 后在服务器执行 `seed`。

## 二、登录阿里云服务器

```bash
ssh root@你的服务器公网IP
```

进入项目目录。你现在的服务器路径如果是 `/root/Final-Killer`，就执行：

```bash
cd /root/Final-Killer
```

如果后续迁移到了 `/opt/final-killer`，就改成：

```bash
cd /opt/final-killer
```

## 三、更新代码版本

先看服务器有没有临时改动：

```bash
git status --short
```

正常情况下应该没有输出。如果有输出，说明服务器上有人改过文件。先不要直接 `pull`，确认这些改动是不是误操作。

备份当前数据库：

```bash
mkdir -p /backup/final-killer
cp server/data/final-killer.db /backup/final-killer/final-killer-$(date +%Y%m%d-%H%M%S).db
```

拉取 GitHub 最新代码：

```bash
git pull --ff-only origin main
```

安装依赖并构建：

```bash
npm ci --include=dev
npm test
npm run build
```

重启服务：

```bash
pm2 restart final-killer --update-env
pm2 status
```

检查接口是否正常：

```bash
curl http://127.0.0.1:4000/api/health
```

正常应看到类似：

```json
{"ok":true,"data":{"status":"ready"},"error":null}
```

## 四、更新题库的正确方式

生产环境已有用户数据后，题库更新不要执行：

```bash
npm run seed
```

原因：`seed` 是首次初始化用的，会重新导入题库并清空或覆盖学习相关数据，可能导致用户进度丢失。

正确方式是使用管理员后台：

1. 打开网站管理员入口。
2. 进入 `/admin/question-bank`。
3. 上传新的 JSON 题库文件。
4. 等系统完成校验。
5. 查看差异预览。
6. 确认无误后点击导入。
7. 导入前系统会自动备份旧版本。
8. 如果导入后发现问题，在同一页面使用历史版本回滚。

题库 JSON 上传流程的作用是：

- 校验格式是否正确。
- 预览新增、修改、删除数量。
- 导入前保存旧版本快照。
- 支持回滚到旧题库。
- 尽量保留未变化题目的学习进度。

## 五、什么时候可以执行 seed

只在这些场景执行：

- 第一次部署，数据库还不存在。
- 测试环境重置数据。
- 你明确接受清空现有学习进度和统计。

已有正式用户数据后，日常更新不要执行：

```bash
npm run seed
```

## 六、常用排查命令

查看 PM2 服务：

```bash
pm2 status
pm2 logs final-killer --lines 100
```

查看端口：

```bash
ss -lntp | grep 4000
```

查看 Nginx 配置是否正确：

```bash
nginx -t
systemctl status nginx
```

重启 Nginx：

```bash
systemctl restart nginx
```

查看 Git 当前版本：

```bash
git log --oneline -5
```

查看当前环境变量文件：

```bash
ls -la .env
```

不要把 `.env` 内容截图发出去，因为里面可能有管理员密码和 JWT 密钥。

## 七、推荐的每次发布检查清单

本地：

- `npm test` 通过。
- `npm run build` 通过。
- `git status --short` 确认只包含本次要提交的改动。
- `git push origin main` 成功。

服务器：

- `git status --short` 没有服务器临时改动。
- 已备份 `server/data/final-killer.db`。
- `git pull --ff-only origin main` 成功。
- `npm ci --include=dev` 成功。
- `npm test` 成功。
- `npm run build` 成功。
- `pm2 restart final-killer --update-env` 成功。
- `curl http://127.0.0.1:4000/api/health` 返回 ready。
- 浏览器访问网站首页、登录页、管理员题库页都正常。

## 八、最短更新命令模板

确认本地已经 push 后，服务器上执行：

```bash
cd /root/Final-Killer
git status --short
mkdir -p /backup/final-killer
cp server/data/final-killer.db /backup/final-killer/final-killer-$(date +%Y%m%d-%H%M%S).db
git pull --ff-only origin main
npm ci --include=dev
npm test
npm run build
pm2 restart final-killer --update-env
pm2 status
curl http://127.0.0.1:4000/api/health
```

如果你的项目路径不是 `/root/Final-Killer`，把第一行改成真实路径。
