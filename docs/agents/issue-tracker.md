# Issue Tracker

This repository uses local markdown issues.

## Location

Write issues under:

```text
.scratch/<feature-or-topic>/
```

Use one markdown file per independently actionable issue. Prefer concise file names such as `login-validation.md` or `review-queue-empty-state.md`.

## Workflow

- Create issues as markdown files.
- Include enough context for an agent to implement without relying on chat history.
- Keep acceptance criteria explicit.
- If this repository later moves to GitHub or GitLab, update this file and `AGENTS.md`.
