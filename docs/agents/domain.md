# Domain Docs

This repository uses a single-context domain layout.

## Sources

- `CONTEXT.md` contains glossary terms only.
- `docs/adr/` is reserved for architectural decisions when they become necessary.

## Consumer Rules

- Read `CONTEXT.md` before changing domain language.
- Do not put implementation details in `CONTEXT.md`.
- Create ADRs only for hard-to-reverse architectural decisions with real trade-offs.
