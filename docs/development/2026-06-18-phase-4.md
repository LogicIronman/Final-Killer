# 期末杀手 Phase 4 开发日志

## 阶段结论

本阶段完成 Phase 4 的第一个扩展功能纵向切片：共享考试日程、主页实时倒计时和管理员日程维护。

完整路径为：

```text
部署者配置管理员账号
  -> 管理员添加多门课程考试时间
  -> 所有登录用户与游客读取共享日程
  -> 主页自动选择最近一场未来考试
  -> 按天、时、分、秒实时倒计时
```

本阶段没有同时铺开题库导入、学习资料和笔记等其他 Phase 4 可选功能。原因是这些能力分别涉及题库版本迁移、文件存储安全和新的用户内容模型；先完成一条可测试、可部署的垂直功能，比同时交付多套半成品更稳定。

## 本阶段目标

1. 提供全站共享的多课程考试日程。
2. 主页展示最近考试的实时倒计时。
3. 多场未来考试可以在主页切换查看。
4. 提供管理员角色和服务端权限边界。
5. 管理员可以添加、修改和删除考试日程。
6. 游客和普通用户只能读取日程，不能修改。
7. 兼容现有 SQLite 数据库，不要求重新 seed。
8. 补齐测试、部署说明、领域术语和 Phase 4 文档。

## 核心设计决策

### 1. 考试日程是全站共享数据

考试时间不是个人学习进度。所有用户和游客读取相同的日程，因此读取接口不要求 JWT：

```text
GET /api/exams
```

创建、修改和删除必须经过管理员鉴权：

```text
POST   /api/exams
PUT    /api/exams/:id
DELETE /api/exams/:id
```

前端只对管理员显示“考试管理”入口，但真正的安全边界位于服务端 `requireAdmin` 中间件。普通用户即使直接调用管理接口，也会收到 HTTP 403。

### 2. 管理员由部署环境配置

新增环境变量：

```text
ADMIN_USERNAME
ADMIN_PASSWORD
```

两项同时设置后，服务启动时会：

- 创建不存在的管理员账号；或
- 将同名账号提升为管理员并同步环境变量中的密码。

这样无需直接修改 SQLite，也不在仓库中内置固定管理员密码。生产环境必须提供独立强密码，不能提交真实 `.env`。

### 3. 主页只倒计时未来考试

倒计时组件会：

1. 过滤已经开始的考试。
2. 按考试时间升序排列。
3. 默认选中最近一场。
4. 每秒更新剩余天、时、分、秒。
5. 存在多场时提供横向切换。

如果当前没有未来考试，页面展示明确空状态；管理员可以直接进入管理页添加日程。

### 4. 动效服务于时间变化

数字切换使用 200ms 的垂直过渡，只表达秒数变化，不增加装饰性页面入场动画。系统启用减少动态效果后，数字直接更新。

## 数据库变化

### 用户角色

`users` 增加兼容字段：

```sql
role TEXT NOT NULL DEFAULT 'user'
```

旧数据库启动时通过 `PRAGMA table_info(users)` 检查字段；缺失时执行非破坏性 `ALTER TABLE`。

### 考试日程

新增：

```sql
CREATE TABLE exam_schedules (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  course_name TEXT NOT NULL,
  exam_at TEXT NOT NULL,
  created_by INTEGER,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL
);
```

同时新增 `exam_at` 索引，保证日程按时间查询。

### 应用设置

新增 `app_settings` 表，用于记录幂等初始化状态。首次迁移会写入一场“马克思主义基本原理”默认考试；后续启动不会重复插入。

本阶段迁移不会删除用户、题目或学习进度，也不需要执行会清空进度的 `npm run seed`。

## 后端实现

### 角色鉴权

- JWT 用户信息包含 `user | admin` 角色。
- `requireAdmin` 先执行登录校验，再检查角色。
- 普通注册始终创建 `user` 角色，客户端不能通过注册请求指定管理员。

### 日程服务

服务层支持：

- 按考试时间和 ID 稳定排序。
- 创建日程并记录创建管理员。
- 修改课程名和考试时间。
- 删除日程。
- 修改或删除不存在记录时返回明确错误。

### 接口输入验证

- 课程名不能为空，最长 80 字符。
- 考试时间必须是包含时区的 ISO datetime。
- 路径 ID 必须是正整数。
- 非法 ID 返回 `VALIDATION_ERROR`。
- 不存在的日程返回 `EXAM_NOT_FOUND` 和 HTTP 404。

## 前端实现

### 主页倒计时

新增懒加载 `ExamCountdown` 组件：

- 展示最近考试名称和时间。
- 使用四格布局展示天、时、分、秒。
- 多场考试可横向切换。
- 管理员显示“管理”快捷入口。
- 无考试时展示可操作空状态。

### 管理员考试管理

新增路由：

```text
/admin/exams
```

页面支持：

- 添加课程名和考试时间。
- 回填表单修改已有日程。
- 页面内二次确认删除，不使用弹窗。
- 加载、保存、错误、空列表和取消编辑状态。
- 非管理员直接访问时展示权限说明。

### 导航和阶段标识

- 管理员导航显示“考试管理”。
- 普通用户和游客不显示管理入口。
- 登录用户顶部阶段标识更新为 `Phase 4`。

## 已知限制

- 当前日程是全站共享，尚不支持用户个人考试计划。
- 已开始的考试仍保留在管理列表，但不会出现在主页倒计时。
- 管理员配置会在每次启动时同步环境变量中的密码；修改环境变量即视为密码轮换。
- 当前没有管理员用户管理、操作审计和多管理员权限分级。
- 题库导入导出、题库切换、学习资料和笔记仍未实现。
- `datetime-local` 表单提交仍建议在真实 Chrome 中完成一次手工验收。
