# 期末杀手 Phase 2 开发日志

## 阶段结论

本阶段在不破坏 Phase 1 登录、游客刷题、进度统计和错题本的基础上，完成了以下四项功能：

1. 刷题页支持返回上一题，并保留本轮选择、判分结果和解析。
2. 键盘支持字母键、数字键和 Enter 连续刷题。
3. 增加重点题列表，覆盖登录用户与游客。
4. 增加“卷狗排行榜”，按注册用户累计提交答案次数排名。

项目文档同步从 Phase 1 MVP 更新到 Phase 2，并补充了产品上下文、设计规范、领域术语和本地待办记录。

## 本阶段需求

### 刷题体验

- 删除原“跳过”按钮，改为“上一题”。
- Enter 在未提交时提交答案，在已经判分后进入下一题。
- 除 `A-E` 外，增加 `1-5` 数字键选择对应顺序的选项。
- 返回上一题时保留选项、正确或错误状态和解析，不重复写入学习进度。

### 新增功能

- 用户可以进入重点题页面查看已标记题目。
- 增加所有注册用户的刷题排行榜。

## 关键设计决策

### 1. 刷题会话状态按题目保存

Phase 1 的刷题页只保存当前题目的 `selected` 和 `result`。切换题目时会清空状态，因此无法支持上一题回看。

本阶段将其改为按题目 ID 保存的会话状态：

```text
题目 ID -> 已选选项 + 判分结果
```

这样返回历史题目时只恢复前端会话快照，不再次调用答题接口，避免重复增加答题次数或重复写入错题历史。

会话快照只负责当前批次的页面回看；登录用户正式学习进度仍以 SQLite 数据库为准，游客正式进度仍以 localStorage 为准。

### 2. Enter 根据当前状态执行唯一动作

键盘流程定义为：

```text
未选择答案 -> Enter 不执行
已选择但未提交 -> Enter 提交答案
已经提交并判分 -> Enter 进入下一题
```

提交过程增加同步锁，并忽略键盘自动连发事件，避免快速长按 Enter 导致重复提交或连续跨题。

### 3. 数字键按选项显示顺序映射

数字键不是固定映射到字母，而是按照当前题目的选项顺序映射：

```text
1 -> 第一个选项
2 -> 第二个选项
3 -> 第三个选项
4 -> 第四个选项
5 -> 第五个选项
```

因此该逻辑也兼容判断题和选项数量不同的题目。

### 4. 重点状态独立于答题状态

重点题不是新的学习进度状态。题目仍然保持 `unseen`、`reviewing` 或 `done`，重点仅由 `is_marked` 表示。

修复了游客模式下的一个边界问题：只标记、尚未答题的题目不能从新题池消失。游客新题筛选现在依据实际答题次数，而不是仅判断 localStorage 中是否存在记录。

### 5. 排行榜不增加冗余数据表

当前用户规模目标为 10-40 人，因此排行榜直接从 `users` 和 `user_progress` 聚合，无需维护额外排行表或缓存。

“刷题量”定义为：

```text
用户所有题目 total_attempts 的累计值
```

排序规则：

1. 累计刷题量降序。
2. 刷题量相同时，累计正确数降序。
3. 前两项都相同时，用户 ID 升序，保证顺序稳定。

游客不参与排名。排行榜只公开用户名、刷题量、正确数和正确率，不返回密码哈希等敏感字段。

## 完成的功能

### 刷题页

- 移除“跳过”按钮。
- 第一题显示禁用的“上一题”。
- 后续题目可以返回上一题。
- 返回后保留所选答案、判分颜色、正确答案和解析。
- 已判分题目不会重复提交。
- `A-E` 选择字母选项。
- `1-5` 选择对应顺序选项。
- Enter 提交或进入下一题。
- 左方向键返回上一题。
- 已判分后右方向键进入下一题。
- 页面快捷键说明已同步更新。

### 重点题

- 新增 `GET /api/progress/marked`。
- 新增 `/marked` 页面。
- 导航栏增加“重点题”。
- 仪表盘“重点题”统计可以点击进入列表。
- 登录用户从 SQLite 查询重点题。
- 游客从 localStorage 查询重点题。
- 列表支持取消重点，取消后立即移出列表。
- 新题接口返回当前题目的重点状态。
- 空状态、加载状态和错误状态已覆盖。

### 卷狗排行榜

- 新增 `GET /api/leaderboard`。
- 新增 `/leaderboard` 页面。
- 导航栏增加“排行榜”。
- 排行榜包含零答题用户。
- 展示排名、用户名、刷题量、正确数和正确率。
- 当前登录用户使用浅蓝背景及“我”标签标记。
- 游客页面说明登录后才能查看和参与排名。
- 移动端表格放入明确的横向滚动容器。

## 数据与接口变化

本阶段没有新增数据库表，也没有执行破坏性迁移。

沿用的字段：

- `user_progress.total_attempts`：排行榜刷题量。
- `user_progress.correct_count`：排行榜累计正确数。
- `user_progress.is_marked`：重点状态。
- `user_progress.status`：未做、复习中、已完成。

新增接口：

| 方法 | 路径 | 鉴权 | 作用 |
| --- | --- | --- | --- |
| GET | `/api/progress/marked` | JWT | 获取当前用户重点题 |
| GET | `/api/leaderboard` | JWT | 获取注册用户排行榜 |

已有接口增强：

- `GET /api/questions/new` 现在同时返回 `isMarked`。
- `POST /api/progress/mark` 语义不变。

## 已知限制

- 独立复习模式页面仍未实现。
- 游客进度不会合并到注册账号。
- 刷题会话回看只在当前批次和当前页面生命周期内保存，刷新页面后会重新抽题。
- 排行榜当前统计所有题库的累计答题次数；项目目前只有一个正式题库。
- 排行榜没有分页。按当前目标用户规模无需分页，用户规模扩大后应增加分页或前 N 名策略。
- 前端暂无组件级或端到端自动化测试。
